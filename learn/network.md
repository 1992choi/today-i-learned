## 개념 정리
### Layer와 Layered 구조의 기본 개념
- Layer란?
  - 레이어(Layer)는 네트워크 통신 기능을 수행하는 기능 단위이다.
  - 하나의 Layer는 특정 기능만을 책임지는 단위이다.
- Layered 구조란?
  - 네트워크 기능을 여러 계층으로 나누어 구성한 구조이다.
  - 하위 계층의 기능을 기반으로 상위 계층이 동작하는 방식이다.
  - 각 계층은 의존 관계로 연결된 구조이다.
    - 상위 계층은 반드시 바로 아래 계층이 제공하는 서비스를 사용하는 구조이다.
    - 예를 들어, L4는 L3, L2, L1이 정상 동작해야 의미를 가지는 구조이다.
- 네트워크에서 Layered 구조가 왜 중요한가?
  - 복잡한 통신을 기능별로 분리하는 목적이다.
  - 모듈화를 위한 구조이다.
  - 표준화된 통신 규칙 적용을 위한 구조이다.

### 네트워크와 네트워킹 그리고 개념
- Network란?
  - Network는 관계의 구조이다.
    - 장치와 장치가 서로 연결되어 관계를 맺고 있는 상태이다.
  - 단순한 연결 선을 의미하는 것이 아니라, 연결을 통해 형성된 구조적 관계를 의미하는 개념이다.
- Networking이란?
  - Networking은 상호작용을 만드는 과정이다.
    - 연결된 장치들이 실제로 데이터를 주고받을 수 있도록 만드는 행위이다.
  - 단순히 연결하는 것이 아니라, 통신이 가능하도록 설정하고 구성하는 활동이다.

### User mode와 Kernel mode
- User mode란?
  - 제한된 권한으로 동작하는 실행 모드이다.
  - 일반 애플리케이션 프로그램이 실행되는 영역이다.
  - 하드웨어 자원에 직접 접근할 수 없는 구조이다.
  - 네트워크 요청을 생성하는 주체가 되는 영역이다.
  - 시스템 콜을 통해서만 Kernel mode에 작업을 요청할 수 있는 구조이다.
- Kernel mode란?
  - 모든 시스템 자원에 접근 가능한 특권 모드이다.
  - 운영체제가 실행되는 영역이다.
  - 메모리, 디스크, 네트워크 장치에 직접 접근할 수 있는 구조이다.
  - 네트워크 스택이 동작하는 핵심 영역이다.
  - User mode의 요청을 받아 실제 처리를 수행하는 구조이다.
- OSI 7계층과 HW / Kernel / User 매핑
  - OSI 7계층은 통신 기능을 나눈 논리적 개념 모델이다.
  - OSI 7계층은 통신 기능을 나눈 논리적 개념 모델이며, HW / Kernel / User 구분은 실제 구현 위치 기준의 구조이지만, 구현 관점에서 다음과 같이 매핑하여 설명할 수 있다.
    - HW 영역
      - OSI 1계층 (Physical)이 해당하는 영역이다.
      - OSI 2계층 일부가 포함되는 구조이다.
        - NIC에서 처리되는 MAC 동작
        - CRC 검사 등 일부 프레임 처리 기능
    - Kernel mode 영역
      - OSI 2계층 일부가 포함되는 구조이다.
      - OSI 3계층 (IP)이 동작하는 영역이다.
      - OSI 4계층 (TCP/UDP)이 동작하는 영역이다.
    - User mode 영역
      - OSI 5 ~ 7계층이 동작하는 영역이다.
      - HTTP, FTP, DNS 등 애플리케이션 프로토콜이 구현되는 영역이다.
      - 브라우저, 서버 프로그램 등 응용 프로그램이 실행되는 구조이다.

### OSI 7계층 주요 프로토콜 정리
- 1계층 (Physical Layer, 물리계층)
  - 전기 신호, 전압, 케이블, 커넥터 등 실제 비트를 전송하는 물리적 매체를 다루는 계층이다.
  - 데이터를 0과 1의 신호 형태로 전달하는 역할을 수행하는 구조이다.
- 2계층 (Data Link Layer, 데이터 링크 계층)
  - Ethernet이 대표적인 프로토콜이다.
  - 같은 네트워크(LAN) 내에서 프레임 단위로 데이터를 전달하는 계층이다.
  - 식별자 : MAC 주소이다.
    - LAN 카드(NIC)를 식별하는 물리적 주소이다.
    - 동일 네트워크 내에서 목적지를 구분하는 기준이 되는 주소이다.
- 3계층 (Network Layer, 네트워크 계층)
  - Internet 계층에 해당하는 영역이다.
    - IP가 대표적인 프로토콜이다.
    - Internet 외에도 다른 프로토콜이 존재하지만, 일반적으로 IP 중심으로 설명하는 구조이다.
  - 서로 다른 네트워크 간 패킷을 전달하는 계층이다.
  - 라우팅을 통해 경로를 결정하는 역할을 수행하는 구조이다.
  - 식별자 : IP 주소이다.
    - 호스트를 식별하는 논리적 주소이다.
    - 네트워크 상에서 목적지를 구분하는 기준이 되는 주소이다.
- 4계층 (Transport Layer, 전송 계층)
  - TCP, UDP 프로토콜이 해당하는 계층이다.
  - 종단 간 통신을 담당하는 계층이다.
  - 데이터의 신뢰성, 흐름 제어, 오류 제어 등을 처리하는 구조이다.
  - 식별자 : Port 번호이다.
    - 하나의 호스트 내부에서 어떤 프로세스와 통신할지 구분하는 식별자이다.
- 5계층 (Session Layer, 세션 계층)
  - SSL(TLS)이 해당하는 영역으로 설명하는 경우가 많다.
  - 통신 세션을 설정하고 유지하는 역할을 수행하는 계층이다.
  - 암호화된 세션을 생성하여 보안을 제공하는 구조이다.
- 6계층 (Presentation Layer, 표현 계층)
  - 데이터 형식 변환, 인코딩/디코딩, 압축, 암호화 등을 담당하는 계층이다.
  - 서로 다른 시스템 간 데이터 표현 방식을 맞추는 역할을 수행하는 구조이다.
- 7계층 (Application Layer, 응용 계층)
  - HTTP가 대표적인 프로토콜이다.
  - 사용자가 직접 사용하는 애플리케이션 프로토콜이 동작하는 계층이다.
  - 웹 브라우저와 웹 서버 간 요청과 응답이 이루어지는 구조이다.

### Host는 이렇게 외우자
- Host란?
  - 간단하게 표현하면, 네트워크에 연결된 computer를 지칭하는 개념이다.
  - 네트워크 상에서 IP 주소를 가지고 통신에 참여하는 장치를 의미하는 용어이다.
  - 데이터를 생성하거나 소비하는 주체가 되는 장치이다.
  - 서버, PC, 스마트폰 등 실제 통신의 끝단에 위치하는 장치가 이에 해당하는 구조이다.

### 스위치가 하는 일과 비용
- 패킷이란?
  - 네트워크에서 전송되는 데이터의 단위이다.
  - 데이터에 출발지/목적지 정보가 포함된 구조이다.
  - 네트워크 장비는 이 정보를 기준으로 데이터를 전달한다.
- Switch란?
  - 네트워크 내부에서 데이터를 목적지 방향으로 전달하는 장치이다.
  - 기본적으로 2계층에서 동작하며, 전달 대상은 프레임이다.
- 역할
  - 수신한 데이터를 분석한다.
  - 목적지 정보를 기준으로 다음 전달 지점을 결정한다.
  - 데이터를 생성하지 않고 중간에서 전달만 수행한다.
- 경로를 선택하는 근거
  - 계층에 따라 기준이 다르다.
    - 2계층 장비: MAC 주소를 기준으로 프레임을 전달한다.
    - 3계층 장비: IP 주소를 기준으로 패킷의 경로를 결정한다.
    - 4계층 이상 장비: 포트 번호 등 추가 정보를 활용하기도 한다.
  - 즉, 어떤 정보를 기준으로 판단하느냐는 장비가 동작하는 계층에 따라 달라진다.
- L3 스위치와 라우터
  - L3 스위치는 3계층(Network Layer)에서 IP 주소를 기준으로 패킷의 경로를 결정한다.
  - 이러한 동작은 라우팅(Routing) 기능과 동일하다.
  - 따라서 기능 관점에서는 L3 스위치를 라우터로 볼 수 있다.
  - 다만, 장비 분류상으로는 스위치 계열 장비에 라우팅 기능이 추가된 형태이다.

### NIC, L2 Frame, LAN 카드 그리고 MAC 주소
- NIC (Network Interface Card)
  - 흔히 LAN 카드라고 부르는 하드웨어 장치이다.
  - 컴퓨터를 네트워크에 연결해주는 물리적 인터페이스이다.
  - NIC은 하드웨어 장치이며, 고유한 MAC 주소를 가진다.
- MAC 주소
  - 2계층(Data Link Layer)에서 사용되는 물리적 주소이다.
  - 일반적으로 48비트(6바이트)로 구성된다.
  - 동일 네트워크(LAN) 내부에서 장치를 식별하는 용도로 사용된다.
  - NIC에 할당되어 출고되며, 필요에 따라 소프트웨어적으로 변경이 가능하다.
- L2 Frame
  - 2계층에서 전송되는 데이터 단위이다.
  - 구성 요소
    - 목적지 MAC 주소, 출발지 MAC 주소, 데이터, 오류 검출 정보 등
  - 동일 네트워크 내에서 스위치가 전달하는 대상이다.

### L2 스위치에 대해서
- L2 스위치란?
  - OSI 2계층(Data Link Layer)에서 동작하는 장비이다.
  - MAC 주소를 기준으로 프레임을 전달한다.
  - 동일 네트워크(LAN) 내부에서 장비 간 통신을 가능하게 한다.
- L2 스위치의 역할 구분
  - L2 스위치는 네트워크 설계 구조에 따라 역할이 나뉜다.
  - 대표적으로 Access와 Distribution 역할로 구분된다.
  - 이는 장비 종류의 차이가 아니라, 배치 위치와 기능 중심의 구분이다.
- L2 Access Switch
  - End-point와 직접 연결되는 위치에 배치된다.
  - MAC 주소를 기준으로 프레임을 스위칭한다.
  - 단말이 네트워크에 접속하는 지점 역할을 수행한다.
  - 스위치와 장비가 정상적으로 연결되어 통신 가능한 상태를 Link-up이라고 한다.
    - 물리적 연결이 끊기거나 통신이 불가능한 상태는 Link-down이라고 한다.
- L2 Distribution Switch
  - 여러 L2 Access 스위치를 상위에서 연결하는 위치에 배치된다.
  - 트래픽을 집선(Aggregation)하는 역할을 수행한다.
  - VLAN(Virtual LAN) 기능을 제공하는 것이 일반적이다.
    - VLAN(Virtual LAN)이란?
      - 하나의 물리적 네트워크를 논리적으로 여러 개의 네트워크처럼 분리하는 기능이다.
      - 서로 다른 VLAN에 속한 장비는 기본적으로 2계층에서 직접 통신할 수 없다.
      - 브로드캐스트 도메인을 분리하는 목적을 가진다.

### LAN과 WAN의 경계 그리고 Broadcast
- Broadcast
  - Broadcast란?
    - 개념
      - Broadcast는 네트워크에 연결된 모든 장비에게 데이터를 동시에 전달하는 통신 방식이다.
      - 특정 대상이 아니라 같은 네트워크에 속한 모든 장비에게 패킷을 전달하는 것이 특징이다.
  - Broadcast의 주소
    - 특별한 주소가 존재함
    - MAC과 IP 모두 존재
      - MAC 주소: FF:FF:FF:FF:FF:FF
      - IP 주소: 네트워크의 Broadcast 주소 (예: 192.168.0.255)
  - 특징
    - 하나의 장비(Host)가 Broadcast를 전송하면 같은 네트워크에 있는 모든 장비가 해당 패킷을 수신한다.
    - Broadcast 트래픽이 많아지면 네트워크 전체 성능에 영향을 줄 수 있다.
    - 따라서 Broadcast의 범위(Broadcast Domain)는 가능한 작게 유지하는 것이 좋다.
    - 하지만 Broadcast 범위를 너무 작게 나누면 필요한 장비까지 패킷이 전달되지 않을 수 있다.
    - 이러한 이유로 네트워크에서는 보통 IP 네트워크 단위(Subnet)를 기준으로 Broadcast 범위를 구분한다.
- LAN과 WAN
  - 강의에서의 설명
    - HW는 물리(Physical) 영역
    - SW는 논리(Logical) 영역으로 표현하기도 한다.
  - LAN
    - 물리적 통신 범위라고 설명하기도 한다.
    - 하나의 건물, 사무실, 데이터센터처럼 물리적으로 가까운 네트워크를 의미한다.
  - WAN
    - 논리적 통신 범위라고 설명하기도 한다.
    - 여러 LAN을 논리적으로 연결한 더 넓은 네트워크를 의미한다.
    - 대표적으로 인터넷이나 통신사 네트워크를 통해 연결되는 구조가 이에 해당한다.
  - 주의할 점
    - LAN = 물리 / WAN = 논리 라는 구분이 공식적인 정의는 아니다.
    - 단지 네트워크 범위를 직관적으로 이해하기 위한 설명 방식이다.

### IPv4 주소의 기본 구조
- IPv4
  - 구조
    - 32bit로 구성된다.
      - = 8bit × 4
      - = 256개의 값 × 4
      - 각 옥텟은 0 ~ 255 범위를 가진다.
    - Ex) 192.168.0.10
      - 2진수(bit)로 표현하면 아래와 같다.
        - `1100 0000 . 1010 1000 . 0000 0000` . `0000 1010`
      - 이 예시에서는 앞 24bit가 Network ID이고, 뒤 8bit가 Host ID이다.
        - `1100 0000 . 1010 1000 . 0000 0000` → Network ID
        - `0000 1010` → Host ID

### L3 IP Packet으로 외워라
- Packet 이란?
  - 네트워크에서 전송되는 데이터의 단위이다.
  - 3계층(Network Layer)에서 다루는 데이터 단위를 Packet이라고 한다.
  - 상위 계층에서 전달된 데이터를 목적지까지 전달하기 위해 주소 정보 등을 포함하여 구성된 데이터 구조이다.
- 구성
  - Packet은 Header와 Payload로 나뉜다.
    - Header
      - 패킷을 전달하기 위해 필요한 제어 정보가 담기는 영역이다.
      - 대표적으로 다음과 같은 정보가 포함된다.
        - 출발지 IP 주소
        - 목적지 IP 주소
        - 프로토콜 정보(TCP, UDP 등)
        - TTL(Time To Live)
      - 네트워크 장비는 Header 정보를 기준으로 패킷을 전달한다.
    - Payload
      - 실제 전달하려는 데이터가 들어 있는 영역이다.
      - 상위 계층(예: TCP/UDP, HTTP 등)에서 전달된 데이터가 포함된다.
- MTU (Maximum Transmission Unit)
  - 한 번에 전송할 수 있는 최대 데이터 크기를 의미한다.
  - 네트워크 장비나 인터페이스가 처리할 수 있는 패킷의 최대 크기이다.
  - 일반적인 Ethernet 환경에서는 MTU가 1500 byte인 경우가 많다.
  - 패킷의 크기가 MTU보다 크면 여러 개의 패킷으로 나누어 전송될 수 있다.

### Encapsulation과 Decapsulation
- 데이터 포함 구조
  - 네트워크 데이터는 계층 구조로 감싸지는 형태로 구성된다.
  - 예를 들어 다음과 같은 포함 관계로 이루어진다.
    - `L2 Frame`은 Header와 Payload로 구성된다. Payload에는 L3 Packet이 포함된다.
      - Header
      - Payload
        - `L3 Packet` (마찬가지로 Payload에 Segment 존재)
          - Header
          - Payload
            - `L4 Segment`
              - Header
              - Payload
                - L5 ~ L7 데이터 (Application Data)
  - 즉, 상위 계층의 데이터가 하위 계층의 Payload 안에 포함되는 구조이다.
- Encapsulation
  - 데이터를 전송할 때 상위 계층 데이터에 하위 계층 Header가 하나씩 추가되는 과정을 의미한다.
  - 각 계층이 자신의 제어 정보를 추가하면서 데이터를 감싸는 구조이다.
  - 결과적으로 상위 계층 데이터가 하위 계층의 Payload 안에 포함된다.
- Decapsulation
  - 수신 측에서 하위 계층부터 Header를 하나씩 제거하면서 데이터를 상위 계층으로 전달하는 과정을 의미한다.
  - 즉, Encapsulation의 반대 과정이다.
  - 네트워크 장비나 호스트는 Header 정보를 확인한 뒤 제거하고 내부 데이터를 다음 계층으로 전달한다.

### 패킷의 생성과 전달
- 패킷은 데이터 통신 과정에서 생성되어 네트워크를 통해 전달되고, 목적지에 도착하면 원래 데이터로 복원된다.
  - 패킷의 생성
    - 사용자가 웹 요청, 메시지 전송 등의 작업을 하면 Application 계층에서 데이터가 생성된다.
    - 이 데이터는 네트워크 계층을 내려가면서 각 계층의 Header가 추가된다.
    - 이 과정을 Encapsulation이라 하며, 최종적으로 L3 Packet 형태가 만들어진다.
  - 패킷의 전달
    - 생성된 패킷은 네트워크 장비(스위치, 라우터 등)를 거쳐 목적지로 전달된다.
    - 라우터는 Packet의 IP 주소를 기반으로 다음 목적지를 결정하여 전달한다.
  - 패킷의 소멸
    - 패킷이 목적지 장치에 도착하면 Decapsulation 과정이 이루어진다.
    - 각 계층의 Header가 제거되면서 원래의 Application 데이터가 복원된다.
    - 최종적으로 해당 데이터는 Application에서 사용된다.

### 계층별 데이터 단위
- L2 (Data Link Layer)
  - 데이터 단위 : Frame
  - 실제 물리 네트워크에서 전달되는 단위이다.
- L3 (Network Layer)
  - 데이터 단위 : Packet
  - IP 주소를 기반으로 목적지까지 전달된다.
- L4 (Transport Layer)
  - 데이터 단위 : Segment
  - 상위 계층의 데이터를 일정 크기로 나누어 전송한다.
  - MSS(Maximum Segment Size)에 의해 한 Segment의 최대 데이터 크기가 결정된다.
- L5 이상 (Application 영역, User 모드)
  - 데이터 단위 : Stream
  - 사용자가 생성한 연속적인 데이터 흐름이다.
  - 이 데이터는 전송을 위해 L4에서 여러 Segment로 나뉘게 된다.
    - 이는 한 번에 전송할 수 있는 데이터 크기가 제한되어 있기 때문이다.
    - L4에서는 MSS(Maximum Segment Size)에 의해 하나의 Segment가 가질 수 있는 최대 데이터 크기가 결정된다.
    - 따라서 MSS보다 큰 데이터는 여러 개의 Segment로 나누어 전송된다.
    - 이러한 과정을 Segmentation이라고 한다.

### 이해하면 인생이 바뀌는 TCP/IP 송·수신 구조
- 구조
  - TCP/IP 통신 구조는 크게 소프트웨어 영역과 하드웨어 영역으로 나누어 볼 수 있다.
    - 소프트웨어
      - User Mode
        - Application이 동작하는 영역
        - 네트워크 통신은 Socket I/O를 통해 커널에 요청한다
      - Kernel Mode
        - 실제 네트워크 통신 처리가 이루어지는 영역
        - TCP
        - IP
    - 하드웨어
      - NIC(Network Interface Card) 등을 통해 실제 네트워크로 데이터가 전송된다.
- 과정
  - 가정
    - 예를 들어 A와 B 두 시스템이 있고, A에서 `profile.png` 파일을 B로 전송한다고 가정한다.
  - 송신 과정
    - Application(User Mode)에서 `profile.png` 파일을 읽는다.
    - 읽은 데이터는 Socket I/O를 통해 Kernel 영역으로 전달된다.
    - 전달된 데이터는 송신 버퍼(Send Buffer)에 저장된다.
    - 파일과 같은 데이터는 한 번에 모두 전송할 수 없기 때문에 일정 크기로 나누어 전송해야 한다.
    - 이때 데이터를 여러 단위로 나누는 과정을 Segmentation이라고 한다.
    - 나뉜 데이터는 TCP Segment 형태로 만들어지고 IP 계층을 거쳐 네트워크로 전송된다.
  - 수신 과정
    - B 시스템에서는 네트워크를 통해 TCP Segment를 수신한다.
    - 수신된 데이터는 Kernel 영역의 수신 버퍼(Receive Buffer)에 저장된다.
    - TCP는 데이터를 정상적으로 받았다는 신호로 ACK를 송신 측에 전송한다.
    - 송신 측은 ACK를 받으면 수신 측이 정상적으로 데이터를 받고 있다고 판단하고 다음 데이터를 계속 전송한다.
    - 수신 버퍼에 쌓인 데이터는 Application(User Mode)이 Socket을 통해 읽어간다.
  - 수신 버퍼와 흐름 제어
    - 수신 측의 Receive Buffer에는 전달받은 데이터가 계속 쌓이게 된다.
    - Application이 데이터를 읽어가는 속도가 느리면 버퍼의 여유 공간이 점점 줄어든다.
    - TCP는 현재 수신 버퍼의 여유 공간을 송신 측에 알려주어 전송 속도를 조절한다.
- 네트워크 통신 중 발생할 수 있는 문제
  - 유실 (Packet Loss)
    - 전송 중 패킷이 네트워크에서 사라지는 현상
  - 재전송 (Retransmission)
    - 송신 측이 일정 시간 동안 ACK를 받지 못하면 패킷이 유실되었다고 판단하고 다시 전송한다
  - DUP ACK
    - 동일한 ACK가 여러 번 도착하는 현상
  - Out of Order
    - 네트워크 경로 등의 이유로 패킷의 도착 순서가 뒤바뀌는 현상
  - Zero Window
    - 수신 측 버퍼에 더 이상 여유 공간이 없어 데이터를 받을 수 없는 상태
    - Application이 버퍼의 데이터를 처리하는 속도가 느릴 때 발생할 수 있다
